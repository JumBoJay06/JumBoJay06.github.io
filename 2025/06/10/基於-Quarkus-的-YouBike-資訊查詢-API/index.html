基於 Quarkus 的 YouBike 資訊查詢 API

Quarkus 入門學習範例

建立專案

  • 可以使用 Quarkus 官方網站的產生器 (code.quarkus.io)。
  • 或者用 IDE 的 Plugin 來建置(e.g, Intellij 的 quarkus)
  • 此範例會用 Maven 來專案管理

設定檔

有 yaml 或是 properties 的檔案格式，預設是用 properties。

若想改用 yaml 要新增依賴

  • 此專案以 yaml 當範例
application.yaml

POM.xml

定義專案的基礎結構、依賴管理和建置方式。

此範例的設置

  • Quarkus 版本: 3.23.0。
  • Java 版本: 21。
  • 核心依賴:
    • quarkus-rest: 提供建立 JAX-RS (Java API for RESTful Web Services) 端點的核心功能。
    • quarkus-rest-jackson: 提供使用 Jackson 函式庫進行 Java 物件與 JSON 之間自動轉換的功能。
    • quarkus-rest-client-jackson: 一個強大的擴充功能，可以輕易地建立類型安全的 REST API 客戶端來呼叫外部服務，同樣整合了 Jackson。
    • quarkus-config-yaml: 讓專案能夠讀取 .yaml 格式的設定檔。
  • 測試依賴:
    • quarkus-junit5 & quarkus-junit5-mockito: 提供 JUnit 5 的測試環境，並整合 Mockito 以便進行單元測試的 Mock（模擬）操作。
    • rest-assured: 一個便於撰寫 REST API 測試的函式庫，語法流暢易讀。
  • 建置與打包:
    • quarkus-maven-plugin: Quarkus 的核心 Maven 插件，負責編譯、打包以及轉換為原生執行檔等任務。
    • native Profile: 專案設定了一個名為 native 的 Maven Profile，當啟用時 (Pnative)，會將專案建置成一個啟動速度極快、記憶體佔用極低的原生執行檔。

程式碼結構與詳細解析

專案遵循了標準的分層架構：Resource (資源層)、Service (服務層)、Client (客戶端層) 和 Model (模型層)。

Model

這是一個 Java record，用來映射從 YouBike API 獲取的每一筆站點資料。

  • record: Java 16+ 的語法糖，可以極簡化地建立不可變的資料傳輸物件 (DTO)，自動產生建構子、equals()、hashCode() 和 toString() 方法。
  • @JsonProperty: 來自 Jackson 函式庫，作用是將 JSON 中的鍵 (key) 映射到 Java record 的欄位上。例如，將 JSON 的 "sno" 映射到 stationNo 欄位。
  • @JsonIgnoreProperties(ignoreUnknown = true): 一個非常有用的註解。如果遠端 API 回傳的 JSON 中包含了 YouBikeStation record 未定義的欄位，Jackson 會預設拋出錯誤。此註解讓 Jackson 忽略這些未知的欄位，增強了程式的健壯性。
完整檔案

Client

這個檔案完美展示了 MicroProfile REST Client 的優雅之處。

  • interface: 開發者只需定義一個介面，Quarkus 會在編譯期間自動為其產生實作。
  • @RegisterRestClient(configKey = "you-bike-api"): 核心註解。它告訴 Quarkus 這是一個 REST Client，並且其相關設定（如 URL）可以在 application.yaml 中以 you-bike-api 為鍵找到。
  • @GET: 指定呼叫此方法時會發送一個 HTTP GET 請求到設定的 URL。
  • List<YouBikeStation> getYouBikeList(): 定義了方法的返回類型。Quarkus 和 Jackson 會自動將收到的 JSON 陣列反序列化成 YouBikeStation 的列表。
完整檔案

Service

服務層封裝了主要的業務邏輯，將資料獲取與處理的細節和 API 層分離。

  • @ApplicationScoped: 一個 CDI (Contexts and Dependency Injection) 註解，表示這個 Service 在整個應用程式的生命週期中只會有一個實例（單例）。
  • @Inject @RestClient YouBikeApiClient youBikeApiClient: CDI 的依賴注入。Quarkus 會將前面定義並自動實作的 YouBikeApiClient 注入到這個欄位中。
  • fetchAllStationsFromExternalApi(): 這是一個私有方法，負責實際呼叫 youBikeApiClient 來獲取外部資料。它也包含了日誌記錄和基本的異常處理（如果發生錯誤，則返回一個空列表）。
  • getAllStations() 和 getStationById(): 提供給 Resource 層呼叫的公開方法。getStationById 使用 Java Stream API 來過濾所有站點，找出符合指定 ID 的站點。
完整檔案

Resource

這是直接對外的 API 端點，負責接收 HTTP 請求並回傳 HTTP 回應。

  • @Path("/api/youbike"): 定義了這個類別中所有 API 的基礎路徑。
  • @Inject YouBikeService youBikeService: 注入服務層的實例，讓 Resource 層可以呼叫其業務邏輯方法。
  • getAllYouBikeStations():
    • 路徑為 @Path("/all")，完整路徑是 /api/youbike/all。
    • 呼叫 youBikeService.getAllStations() 獲取資料。
    • 如果列表為空，回傳 HTTP 404 Not Found 狀態碼和錯誤訊息；否則回傳 200 OK 和站點列表。
  • getYouBikeStationById():
    • 路徑為 @Path("/station/{id}")，其中 {id} 是動態路徑參數。
    • @PathParam("id"): 將 URL 中的 {id} 值綁定到 stationId 方法參數上。
    • 輸入驗證: 檢查 stationId 是否為空或空白，如果是，則回傳 400 Bad Request，防止無效查詢。
    • 優雅處理: 使用 Optional 來處理查詢結果。如果 youBikeService 回傳的 Optional 有值，則回傳 200 OK；如果為空，則回傳 404 Not Found。
完整檔案

測試

測試是確保程式碼品質的關鍵。

  • @QuarkusTest: 啟動一個輕量級的 Quarkus 執行環境來進行測試，讓測試的環境與實際運行環境非常接近。
  • @InjectMock YouBikeService youBikeService: 這是測試的精髓。它不會注入真正的 YouBikeService（那樣會發出真實的網路請求），而是注入一個 Mockito 的模擬物件。這讓測試可以獨立於外部 API，變得快速且穩定。
  • 測試案例:
    • test..._Success: 模擬 youBikeService 成功回傳資料，並使用 RestAssured 驗證 API 是否回傳 200 OK 且內容正確。
    • test..._NotFound: 模擬 youBikeService 回傳空資料或 Optional.empty()，驗證 API 是否回傳 404 Not Found。
    • test..._BadRequest: 直接測試 API 的輸入驗證邏輯，傳入一個空白 ID，驗證是否回傳 400 Bad Request。

請求流程追蹤 (以 GET /api/youbike/station/500101001 為例)

  1. 使用者發出 HTTP GET 請求。
  2. Quarkus 接收請求，根據路徑 @Path 路由到 YouBikeResource 的 getYouBikeStationById 方法。
  3. @PathParam 將 “500101001” 注入 stationId 變數。
  4. YouBikeResource 呼叫已注入的 youBikeService.getStationById("500101001")。
  5. YouBikeService 呼叫 youBikeApiClient.getYouBikeList()。
  6. Quarkus 的 REST Client 根據 application.yaml 的 URL，向 https://tcgbusfs.blob.core.windows.net/... 發送一個真正的 HTTP GET 請求。
  7. 收到遠端的 JSON 回應後，Jackson 自動將其轉換為 List<YouBikeStation>。
  8. YouBikeService 使用 Stream 過濾這個 List，找到 stationNo 為 “500101001” 的物件，並將其包裝在 Optional 中回傳。
  9. YouBikeResource 收到 Optional，因為它有值，所以進入 .map() 邏輯，將 YouBikeStation 物件再次透過 Jackson 序列化成 JSON 字串。
  10. YouBikeResource 建構一個 200 OK 的 Response 物件並回傳給使用者。
