打造一個功能齊全的 React Native 天氣 App：從 API 串接到狀態管理

發表於 2025-07-16 分類於 程式語言 ， React Native ， Expo ， Redux ， Fetch ， async-storage Disqus：

打造一個功能齊全的 React Native 天氣 App：從 API 串接到狀態管理

你好！在這篇文章中，我們將一步步帶你使用 React Native 和 Expo，從頭開始打造一個天氣 App。這個 App 不僅能讓你查詢任何城市的天氣，還會儲存你的查詢紀錄，方便你快速查看。

這個專案將會涵蓋幾個重要的學習重點：

  • API 串接：使用 fetch 來獲取 OpenWeather API 的天氣資料。
  • 非同步操作：靈活運用 useEffect 和 useState 來處理非同步的資料請求。
  • 本地儲存：透過 AsyncStorage 將查詢歷史紀錄儲存在手機本機。
  • 狀態管理：使用 Redux 這樣的狀態管理工具。
  • 頁面導覽：使用 Stack Navigator 來管理搜尋頁與歷史紀錄頁之間的切換。
  • 列表與狀態顯示：運用 FlatList 和 ScrollView 來優化列表顯示，並處理載入中（Loading）和無資料的提示。

這是我們的最終目標：

  • 🔍 搜尋城市頁：輸入城市並查詢天氣
  • 📜 歷史紀錄頁：儲存並顯示最近查過的城市

讓我們開始吧！

專案結構概覽

我們的天氣 App 結構如下：

串接 OpenWeather API

要取得天氣資訊，我們首先需要與提供資料的 API 進行溝通。在這個專案中，我們使用 OpenWeather API。

  1. 設定 API 金鑰和 URL
    • 在環境變數中新增 API 金鑰:
      • 在跟目錄中新增.env檔案
      • 在檔案中輸入EXPO_PUBLIC_WEATHER_API_KEY="你的金鑰"
    • src/api/weatherAPI.ts檔案中統一管理所有 API 請求的邏輯:
      • 首先，我們從環境變數中讀取 API 金鑰，並定義 API 的網址。這麼做可以避免將敏感資訊直接寫在程式碼中。我們定義了兩個 API 端點：一個用來透過城市名稱取得經緯度，另一個用來透過經緯度取得詳細的天氣資訊。
  2. 建立 API 請求函式
    我們使用 fetch 來發送網路請求。fetchWeatherDataAPI 函式會先呼叫 API_2_5_URL 取得經緯度，再將經緯度傳給 fetchWeatherDataAPIByLocation 函式來取得完整的天氣資料。這樣拆分的好處是，當使用者從歷史紀錄點擊時，我們可以直接使用已儲存的經緯度查詢，減少一次 API 請求。

使用 Redux Toolkit 管理 App 狀態

對於需要跨頁面共享的狀態（如天氣資料、歷史紀錄），使用 Redux 這樣的狀態管理工具會比單純用 useState 和 props 傳遞來得更有效率。

我們在 src/redux/weatherSlice.ts 檔案中定義了所有與天氣相關的狀態和操作。

  1. 建立非同步 Thunks
    createAsyncThunk 是 Redux Toolkit 提供的一個強大工具，它能幫我們輕鬆處理非同步請求，並自動產生 pending, fulfilled, rejected 這三種狀態的 action。

    我們建立了幾個 thunks：

    • fetchWeather: 處理使用者輸入城市名稱的查詢。
    • fetchWeatherByHistory: 處理從歷史紀錄點擊後的查詢。
    • loadHistory: 從 AsyncStorage 讀取歷史紀錄。
    • saveHistory: 將新的查詢結果存入 AsyncStorage。
    • clearHistory: 清除所有歷史紀錄。

  2. 建立 Slice
    createSlice 會幫我們整合 reducer 和 action creators。我們在 extraReducers 中處理由 createAsyncThunk 產生的 actions，並根據不同的請求狀態（等待、成功、失敗）來更新我們的 state，例如設定 loading 狀態或儲存錯誤訊息。
    如果有一些不用非同步的需求，可以寫在 reducer 裡，但這次需求都是需要非同步的，就沒有寫範例了

    最後，在 src/redux/store.ts 中設定好我們的 store，並在 App.tsx 中用 <Provider> 將 store 提供給整個 App。

打造畫面與互動邏輯

搜尋頁面 (SearchScreen.tsx)

這是我們的 App 首頁，包含了輸入框、搜尋按鈕和天氣資訊顯示區塊。

  • 狀態與互動：我們使用 useState 來管理輸入框的文字。useDispatch 和 useSelector 則用來與 Redux store 互動，發送 action 並取得最新的天氣資料、載入狀態和錯誤訊息。
  • 處理搜尋：點擊搜尋按鈕時，會呼叫 handleSearch 函式。這個函式會先透過 cityNameMap 將使用者輸入的中文地名轉換成 API 需要的英文格式，然後 dispatch fetchWeather 這個 thunk。
  • 顯示不同狀態：根據從 Redux store 取得的 loading 和 er``ror 狀態，畫面會顯示對應的 UI：
    • loading === 'pending': 顯示 <ActivityIndicator /> 轉圈動畫。
    • error: 顯示錯誤訊息。
    • loading === 'succeeded': 顯示天氣資訊。
  • 從歷史紀錄頁面返回：我們使用 useEffect 來監聽 route.params 的變化。當使用者從歷史紀錄頁點擊一個城市返回時，這個頁面會接收到傳來的 locationData，並自動觸發一次新的查詢。
SearchScreen.tsx 完整檔案

歷史紀錄頁 (HistoryScreen.tsx)

這個頁面用來顯示使用者過去的查詢紀錄。

  • 載入歷史：我們使用 useEffect 搭配 navigation.addListener('focus', ...)，確保每次進入這個畫面時，都會 dispatch loadHistory 來從 AsyncStorage 取得最新的歷史紀錄。
  • 顯示列表：FlatList 是顯示長列表的最佳選擇，它有很好的效能優化。我們將從 Redux store 取得的 history 陣列作為 data 傳給它。如果沒有任何歷史紀錄，ListEmptyComponent 會顯示提示文字。
  • 清除歷史：點擊「清除所有紀錄」按鈕會跳出一個 Alert 確認視窗，使用者確認後，會 dispatch clearHistory action 來清空 AsyncStorage 和 Redux state 中的歷史紀錄。
