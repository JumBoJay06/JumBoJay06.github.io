Compose Multiplatform 建立共用資源檔案

我們在上一篇已經完成了一個 App，這次我們再來進階一點，建立共用資源檔，讓 App 更加完整。

這次目標大致上有兩個：

  • 建立語系檔
  • 建立 App Theme

語系檔

你可能想說，我只有一個語系也要建立嗎？
建議還是要有比較好喔，有了語系檔可以讓統一管理全部的字串，避免文檔四散各地不好管理

怎麼建立

首先，共用的資源檔都會在composeApp/src/commonMain/composeResources/裡，會依照檔案性質區分資料夾，如 drawable、values等等，有時候不會預設幫你建立資了夾，這時自己手動新增就可以了。
這邊就先專注在如何建立語系檔吧～

  • values
    我們的語系檔會在這個資料夾之下

建立語系

語系檔會在valus之下，那要怎麼區分各個語系呢～
首先，values你可以當作是預設的設置，沒支援到的語系都會以values為主。
建立語系會以資料夾來區分，資料夾的命名規則是values-<language_code>_<country_code>，其中<language_code>是 ISO 639-1 的語言代碼，<country_code>是 ISO 3166-1 alpha-2 的國家代碼，可以不用加。

  • 對於英文，資料夾名稱就是values-en。
  • 對於日文，資料夾名稱就是values-ja。
  • 對於韓文，資料夾名稱就是values-ko。
  • 對於中文(台灣)，資料夾名稱就是values-zh_TW。

App Theme

MaterialTheme 是 Jetpack Compose 中實作 Google Material Design 設計系統的核心。簡單來說，它是一個集中管理你 App 整體外觀和風格的框架，讓你的 UI 擁有一致的視覺表現。

你可以將 MaterialTheme 想像成一個包裹在你 App UI 外層的「主題設定器」。所有放在它裡面的 Composable（可組合項），例如按鈕、文字、卡片等，都會自動套用你定義好的風格。

MaterialTheme 的三大核心元素

MaterialTheme 主要由以下三個部分構成:

  1. 色彩配置 (Color Scheme)
    這是定義 App 顏色的地方。Material Design 3 (M3) 定義了一系列具有特定語義的顏色角色，而不是像過去一樣只用 primary, secondary。例如：

    • primary: 主要互動元素（如按鈕、輸入框焦點）的顏色。
    • surface: UI 元件「表面」的顏色，如卡片、對話框的背景。
    • background: 整個畫面的背景色。
    • onPrimary, onSurface, onBackground: 在上述顏色「之上」的內容（通常是文字或圖示）所使用的顏色，以確保對比度和可讀性。

    透過 MaterialTheme.colorScheme，可以在任何 Composable 中輕鬆取用這些預先定義好的顏色。

  2. 排版 (Typography)
    這裡定義了 App 中所有文字的樣式，例如字體、大小、粗細和行高等。Material Design 提供了一套排版層級，例如：

    • displayLarge, displayMedium, displaySmall
    • headlineLarge, headlineMedium, headlineSmall
    • titleLarge, titleMedium, titleSmall
    • bodyLarge, bodyMedium, bodySmall
    • labelLarge, labelMedium, labelSmall

    然後在需要的地方透過 MaterialTheme.typography 來套用，例如 Text("Hello", style = MaterialTheme.typography.titleLarge)。這能確保整個 App 的文字風格一致。

  3. 形狀 (Shapes)
    這裡定義了 UI 元件的邊角形狀，例如按鈕、卡片、對話框的圓角大小。可以定義不同尺寸的形狀：

    • extraSmall, small, medium, large, extraLarge

    元件會根據其大小和用途自動選用合適的形狀。例如，一個 Card 可能會使用 medium 的圓角，而一個 FloatingActionButton 可能會使用 large 的圓角。可以透過 MaterialTheme.shapes 來取用這些設定。
