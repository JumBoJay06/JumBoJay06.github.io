用 React Native 和 Expo 打造你的第一個多頁待辦事項 App

剛接觸一個新的框架或技術時，親手打造一個應用程式是最快的學習方式。今天，我將帶大家一步步使用 React Native 從零開始，建立一個功能完整、包含多個頁面的 Todo List 應用程式。

完成後，你的 App 將會有兩個頁面：一個是待辦事項的清單頁，你可以在這裡新增、刪除、標記完成；另一個是詳細內容頁，點擊清單中的任何一項就可以進入編輯。

這是我們的最終目標：

  • 清單頁 (List): 顯示所有待辦事項，並可新增。
* 詳細頁 (Detail): 顯示單一事項的詳細內容，並可編輯。

讓我們開始吧！

🎯 學習重點

在我們深入探索程式碼之前，首要任務是設定好我們的開發環境，並了解專案需要哪些「武器」（也就是所謂的依賴套件）。這是一個從無到有啟動專案的完整流程。

安裝 Node.js 與 npm

npm (Node Package Manager) 是 JavaScript 世界的標準套件管理工具，我們將用它來安裝和管理專案所需的所有函式庫。npm 會跟著 Node.js 一起安裝。

  • 前往 Node.js 官網：https://nodejs.org/
  • 下載並安裝：建議下載「LTS」（Long-Term Support，長期支援）版本，它的穩定性最高。安裝過程非常簡單，一直按下一步即可。

安裝完成後，打開你的終端機（Terminal）並執行以下指令，確認是否安裝成功。如果能看到版本號，就代表成功了。

安裝 Expo CLI

Expo CLI 是 Expo 的命令列工具，讓我們可以快速建立和啟動 React Native 專案。
在終端機中執行以下指令來進行全域安裝：
npm install expo-cli

建立一個新的 Expo 專案

建立我們 Todo List App 的專案。
npx create-expo-app --template
Expo 會引導你選擇一個模板，你可以選擇 blank (TypeScript) 模板開始一個乾淨的專案，再來輸入專案名稱，這樣專案就會建立起來了

專案依賴

package.json 檔案是我們專案的「身分證」，它記錄了專案的名稱、版本、腳本指令，以及最重要的——專案的依賴套件。
打開 package.json，我們來看看 dependencies 區塊：

這些套件是我們 App 能順利運作的基石：

  • react & react-native: React Native 的核心。
  • expo: Expo 框架的核心套件。
  • @react-navigation/native & @react-navigation/native-stack: 用來處理頁面導航和跳轉。
  • @react-native-async-storage/async-storage: 提供一個簡單的、非同步的、持久化的 Key-Value 儲存系統，我們用它來儲存待辦事項，讓使用者下次打開 App 時資料還在。
  • nanoid: 一個小巧的函式庫，用來產生獨一無二的 ID，非常適合用來當作每一筆待辦事項的 id。
  • @expo/vector-icons: Expo 內建的圖示庫，我們用它來顯示新增和刪除的圖示。
  • react-native-screens & react-native-safe-area-context: react-navigation 的底層依賴，用來處理原生導航元件和螢幕的安全區域（例如避開 iPhone 的「瀏海」）。

依賴總結

在這次的實作中，我們將會掌握以下幾個 React Native 開發的關鍵技能：

  1. Expo 環境建置：我們將使用 Expo 這個強大的工具鏈，讓你不需要原生開發經驗也能輕鬆開始。
  2. React 元件化：學習如何使用 JSX 語法，以及 State 和 Props 的概念來建立可重複使用的 UI 元件。
  3. 常用核心元件：熟悉 Text、TextInput、View、FlatList 等基本元件的使用。
  4. 頁面導航 (Navigation)：使用 react-navigation 套件來建立多頁面應用，並在頁面之間切換。
  5. 狀態管理與傳遞：學習如何透過 React Context API 集中管理 App 的狀態，並在不同頁面間傳遞資料。

📂 專案結構概覽

在開始寫程式碼之前，先來看看我們專案的結構：

  • screens/：放置我們應用程式中的主要頁面。
  • components/：放置可在不同頁面中重複使用的 UI 元件。
  • context/：用來處理跨元件的狀態共享問題。
  • styles/：放置共用的樣式設定。

一、用 Context API 進行全域狀態管理

當我們的 App 有多個頁面需要共享或修改同一份資料時（例如我們的待辦事項列表），把這份資料放在一個所有頁面都能存取的地方就非常重要。這就是「全域狀態管理」的概念。我們將使用 React 內建的 Context API 來實現。
讓我們來看看 context/TodoContext.tsx 檔案：

  1. 建立 Context:
    我們使用 createContext 創建了兩個 Context：
  • TodoContext：用來存放我們的 todos 陣列資料。
  • TodoDispatchContext：用來傳遞可以修改資料的 dispatch 函式。
  1. 定義 Reducer:
    todosReducer 是一個純函式，它接收目前的 todos 狀態和一個 action 物件，然後根據 action 的類型回傳一個 新的 狀態。這確保了狀態的變更是可預測的。
    我們定義了幾種操作 (action)：
    • ADD_TODO: 新增一筆待辦事項。
    • UPDATE_TODO: 更新已存在的事項內容。
    • DELETE_TODO: 刪除一筆事項。
    • TOGGLE_TODO: 切換事項的完成狀態。
  2. 提供 Context (Provider):
    TodoProvider 是一個元件，它的作用是包裹住我們的整個 App。它內部使用 useReducer 來管理 todos 的狀態，並透過 useEffect 將 todos 陣列的任何變更都即時儲存到手機的 AsyncStorage 中，這樣即使關閉 App，資料也不會消失。
  1. 自訂 Hooks:
    為了讓其他元件可以方便地取得 todos 資料和 dispatch 函式，我們建立了兩個自訂的 Hook：useTodos 和 useTodosDispatch。
完整檔案

二、建立 UI 元件

待辦清單頁 (screens/TodoListScreen.tsx)

  1. 取得狀態與操作函式: 我們使用剛才建立的自訂 Hook，輕鬆地從全域狀態中取得 todos 清單和 dispatch 函式。
  1. 渲染清單: 我們使用 React Native 提供的 FlatList 元件來顯示待辦事項。FlatList 非常適合長列表，因为它只會渲染螢幕上可見的項目，效能很好。
    • data: 指定要渲染的資料來源，也就是我們的 todos 陣列。
    • renderItem: 這個屬性接收一個函式，告訴 FlatList 每一項應該長什麼樣子。我們在這裡使用 TodoItem 這個獨立元件。
    • keyExtractor: 告訴 FlatList 如何為每個項目產生唯一的 key。
  1. 處理使用者操作
    • 新增: 當使用者在 TextInput 中輸入文字並按下新增按鈕時，我們會呼叫 dispatch，並傳入一個 { type: ACTION_TYPES.ADD_TODO, payload: newTodo }的 action。
    • 刪除/切換狀態: 這些操作被放在了 TodoItem 元件中，但邏輯是相同的：呼叫 dispatch 並傳入對應的 action。
    • 導航: 當使用者點擊一個 TodoItem 時，我們使用 navigation.navigate 函式跳轉到詳細頁，並把該項目的 id 作為參數傳過去。
完整檔案

可重複使用的項目元件 (components/TodoItem.tsx)

這個元件負責顯示單一筆待辦事項。

它接收 item (事項資料) 和幾個函式 (onToggle, onDelete, onNavigate) 作為 props。這種設計讓 TodoItem 元件本身非常單純，它只負責顯示 UI 和把使用者的操作傳遞出去，而不需要知道背後的複雜邏輯。

我們也使用了條件式樣式，如果 item.isComplete 為 true，就顯示一個打勾的圖示，並在文字上加上刪除線。

完整檔案

詳細內容頁 (screens/DetailScreen.tsx)

當使用者從清單頁點擊某個項目時，就會來到這個頁面。

  1. 取得參數: 我們可以從 route.params 中取得從上一個頁面傳過來的 todoId。
  2. 尋找資料: 使用 todoId 從 useTodos() 取得的全域 todos 陣列中，找到我們需要編輯的那筆資料。
  3. 更新標頭: 我們使用 useLayoutEffect 這個 Hook，它會在畫面佈局計算完成後同步執行。我們在這裡動態地設定了頁面標頭，加入一個「儲存」按鈕。
  4. 儲存變更: 當使用者點擊「儲存」按鈕時，我們會 dispatch 一個 UPDATE_TODO 的 action，將修改後的文字和內容傳遞給 reducer，然後使用 navigation.goBack() 返回上一頁。
完整檔案

統一樣式 (styles/commonStyles.tsx)

為了讓我們的 App 看起來更一致，我們把一些共用的樣式，例如卡片陰影效果，抽出來放到 styles/commonStyles.tsx 中。這樣可以方便地在多個元件中重複使用。

完整檔案

導航設定 (App.tsx)

現在我們的 App 不再只有一個頁面了。react-navigation 是 React Native 社群中最受歡迎的導航解決方案。
這是我們設定整個 App 導航結構的地方:

  • NavigationContainer: 這是所有導航器的根容器，必須包裹在整個 App 的最外層。
  • createNativeStackNavigator: 我們使用它來建立一個 Stack (堆疊) 形式的導航器。你可以想像成一疊卡片，新的頁面會被推到這疊卡片的頂部，返回時則是把最上面的卡片移走。
  • Stack.Screen: 我們用它來定義每一個頁面。name 屬性是這個頁面的唯一識別名稱，component 是這個頁面對應的元件。我們也透過 options 屬性來客製化頁面的標頭，例如設定標題文字。
完整檔案

總結

恭喜你！你已經完成了整個 Todo List App 的學習。
我們回顧一下學到的重點：

  • 元件化: 將 UI 切割成可管理的、可重複使用的小塊。
  • 狀態管理: 使用 React Context 和 Reducer 來管理跨頁面的共享資料。
  • 導航: 使用 React Navigation 在不同頁面之間跳轉並傳遞參數。
  • 持久化儲存: 使用 AsyncStorage 讓你的資料可以被保存下來。
