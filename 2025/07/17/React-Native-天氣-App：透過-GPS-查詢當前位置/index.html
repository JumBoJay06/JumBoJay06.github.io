React Native 天氣 App：透過 GPS 查詢當前位置

我們上次完成了天氣查詢 App，這次我們來做小小的擴充，新增查詢當前位置功能，讓 App 更完整吧。

這個專案將會涵蓋幾個重要的學習重點：

  • 獲取當前 GPS: 使用 expo-location 來取得位置權限跟 GPS

這是我們的最終目標：

  • 🔍 搜尋城市頁：新增查詢當前位置按鈕

安裝 expo-location

首先，我們需要安裝 Expo 提供的官方定位套件。請在您的專案根目錄下執行以下指令：
npx expo install expo-location

新增 Redux Thunk (weatherSlice.ts)

現在，我們在 weatherSlice.ts 中新增一個 fetchWeatherByGPS 的 thunk，它會處理向使用者請求定位權限、取得座標，然後觸發天氣查詢的完整流程。

  • 我們新增了一個名為 fetchWeatherByGPS 的 createAsyncThunk。
  • 在這個 thunk 中，我們首先使用 Location.requestForegroundPermissionsAsync() 向使用者請求定位權限。
  • 如果使用者同意，就用 Location.getCurrentPositionAsync({}) 取得目前的經緯度。
  • 最後，將座標傳給 fetchWeatherDataAPIByLocation 來取得天氣資訊。
  • 我們也為這個新的 thunk 在 extraReducers 中加入了 pending, fulfilled, rejected 的處理邏輯，與既有的查詢流程一致。
weatherSlice.ts 完整檔案

更新搜尋頁面 (SearchScreen.tsx)

最後，我們來修改搜尋頁面，加入新的按鈕和對應的處理函式。

  • 我們從 @expo/vector-icons 額外引入了 MaterialCommunityIcons 來使用 crosshairs-gps 圖示。
  • 在輸入框右側，我們新增了一個 Pressable 元件作為 GPS 定位按鈕，並為它設定了新的樣式 iconButton。
  • 按鈕的 onPress 事件會呼叫 handleSearchByGPS，進而 dispatch 我們剛剛建立的 fetchWeatherByGPS action。
  • 我們也微調了 handleSearchAction 的邏輯，確保查詢當前位置不會被記錄到歷史記錄中。
SearchScreen.tsx 完整檔案

總結

現在，當您重新啟動 App 後，就會在搜尋框旁邊看到一個新的 GPS 定位圖示。點擊它，App 就會請求定位權限，並顯示您目前位置的天氣資訊了！
