Compose Multiplatform 介紹

介紹過 Flutter 跟 React Native 了，怎麼能少 Compose Multiplatform，這次我們就用 Compose Multiplatform 寫一個待辦事項的 App 吧。

在開始前，你可能也有聽過 Kotlin Multiplatform，這兩個有什麼差別呢？

技術 負責層面 功用
Kotlin Multiplatform (KMP) 邏輯層 (Logic Layer) 共享應用程式的商業邏輯、資料處理等非 UI 相關的程式碼。
Compose Multiplatform 介面層 (UI Layer) 共享應用程式的使用者介面。

當這兩者結合使用時，開發者可以達到近乎 100% 的程式碼共享。整個專案的架構會非常清晰：

  1. 共享模組 (Shared Module)： 使用 Kotlin Multiplatform 開發所有平台通用的商業邏輯和資料處理。
  2. 共享 UI： 在共享模組中，使用 Compose Multiplatform 來編寫跨平台的 UI 介面。
  3. 平台特定程式碼： 針對需要與底層硬體或作業系統深度互動的功能（例如特定的感測器或平台獨有的 API），再編寫少量的平台特定程式碼。

總結來說，您可以將 Kotlin Multiplatform 視為一個強大的基礎，讓您的應用程式擁有一致且可靠的核心。而 Compose Multiplatform 則是建立在這個基礎之上的一項利器，讓您能用最有效率的方式，為您的應用程式打造出在各個平台上都美觀且一致的使用者體驗。

建立你的第一個 Compose Multiplatform 專案

如何建立專案

IDE 一般來說會選擇 IntelliJ IDEA 或者 Android Studio，以下會以 Android Studio為主

  • Kotlin Multiplatform Wizard: 使用 JetBrains 官方提供的網頁版專案精靈來建立專案
  • Compose Multipatform: 使用 IDE 的 Plugin 來建立專案

這兩者都可以建立專案，可以選一個你方便的來用。

設定專案

  • Project Name: KmpTodoList (或任何你喜歡的名字)
  • Project ID: com.example.kmptodolist (package name)
  • Platforms: 勾選 Android, iOS 等等。

專案結構解析

打開專案後，你會看到這樣的結構：

  • /build.gradle.kts
    • 用途：這是根專案的 Gradle 建構腳本。它的主要作用是為所有子專案（模組）定義通用的 Gradle 插件版本。
    • 內容分析：透過 plugins { ... } apply false 的語法，它在此處宣告了專案會用到的插件（如 Android Application, Kotlin Multiplatform 等），但並不在根專案中實際應用它們。這樣做可以讓所有子模組共享相同的插件版本，避免版本衝突。
  • /settings.gradle.kts
    • 用途：這是 Gradle 的設定檔，用於定義專案的結構。
    • 內容分析：它設定了根專案的名稱為 KmpTodoList，並透過 include(":composeApp") 將 composeApp 這個模組納入專案中。它也設定了依賴解析時要從哪些倉庫（如 google(), mavenCentral()）去尋找函式庫。
  • /gradlew, /gradlew.bat, /gradle/wrapper/gradle-wrapper.properties
    • 用途：這些是 Gradle Wrapper 的檔案。它們確保了無論在哪台電腦上建構這個專案，都會使用 gradle-wrapper.properties 中指定的、完全相同的 Gradle 版本 (gradle-8.13-bin.zip)。這大大提升了建構的一致性。
  • /gradlew/libs.versions.toml
    • 用途：管理全部的依賴，非必要但是建議使用，有助於保持版本一致性及簡化 build.gradle.kts 文件。
  • composeApp 模組
    這是 KMP 專案的核心，包含了絕大部分的程式碼。
    • /composeApp/build.gradle.kts
      • 用途：composeApp 模組的 Gradle 建構腳本，是整個專案最核心的設定檔。
      • 內容分析：
        • Plugins: 實際應用了 KMP、Compose 等插件。
        • Kotlin 平台目標: 定義了專案要建構哪些平台的目標，包括 androidTarget 和 ios (iosX64, iosArm64, iosSimulatorArm64)。
        • Source Sets: 為 commonMain、androidMain、iosMain 等源集分別定義了它們的依賴函式庫。
        • Android 設定: 包含了 Android App 的 namespace、compileSdk、minSdk、applicationId 等標準設定。
    • commonMain (共享程式碼)
      這是所有平台（Android, iOS）共享的程式碼。
    • androidMain (Android 專屬程式碼)
    • iosMain (iOS 專屬程式碼)
  • iosApp 模組
    這是包裝 composeApp 的 Xcode 專案，使其能在 iOS 上運行。
