Flutter 內建的狀態管理方法

發表於 2025-05-13 分類於 程式語言 ， Flutter ， 狀態管理 Disqus：

內建的狀態管理方法

這些是 Flutter SDK 自帶的，適合簡單場景或作為理解更複雜工具的基礎。

StatefulWidget 和 setState()

  • 說明：這是最基本、最直接的狀態管理方式。當 StatefulWidget 的內部狀態改變時，呼叫 setState() 會通知 Flutter 框架該 Widget 的 build() 方法需要重新執行以更新 UI。
  • 適用場景：管理 Widget 內部的、局部的狀態，例如一個 checkbox 是否被選中，或者一個動畫的當前值。
  • 優點：簡單易懂，無需額外套件。
  • 缺點：共用狀態會變得困難，當多個 Widget 需要共享和修改同一個狀態時，程式碼會變得複雜且難以維護。

InheritedWidget

  • 說明：InheritedWidget 是一種特殊的 Widget，它可以將資料有效地從 Widget 樹的父節點向下傳遞到子節點。當 InheritedWidget 更新時，依賴它的子 Widget 會被重建。
  • 適用場景：在 Widget 樹中跨多個層級共享資料，例如應用程式的主題 (Theme)、本地化設定 (Locale) 或使用者認證狀態。
  • 優點：高效的資料傳遞方式。
  • 缺點：直接使用 InheritedWidget 樣板程式碼較多，且狀態的修改邏輯需要自行處理。許多更高級的狀態管理工具 (如 Provider) 底層都使用了 InheritedWidget。

ValueNotifier / ChangeNotifier 搭配 ValueListenableBuilder / AnimatedBuilder / ListenableBuilder

  • ChangeNotifier：一個簡單的類別，它提供變更通知 API (使用 notifyListeners()）。你可以擴展它或混入 (mixin) ChangeNotifier 來管理狀態。
  • ValueNotifier：一個特殊的 ChangeNotifier，它只持有一個值，並在值改變時通知監聽者。
  • ValueListenableBuilder：一個 Widget，它監聽 ValueListenable (如 ValueNotifier) 並在值改變時重建其子樹。
  • AnimatedBuilder / ListenableBuilder：更通用的 Widget，可以監聽任何 Listenable (如 ChangeNotifier)。
  • 適用場景：比 setState() 更進一步，可以將狀態邏輯與 UI 分離，適合中等複雜度的狀態共享。
  • 優點：相對簡單，是 Provider 等套件的基礎概念之一。
  • 缺點：需要手動管理 ChangeNotifier 的生命週期 (例如 dispose)，對於大型應用，依賴注入和狀態管理職責劃分仍需自行設計。
