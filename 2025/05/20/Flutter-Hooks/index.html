Flutter Hooks

Hooks

是一種用來管理 Widget 生命周期的新對象，以減少重覆代碼、增加組件間覆用性。

主要目標和作用

  • 減少樣板程式碼
    • Widget 繼承自 HookWidget (而不是 StatelessWidget)，然後在它的 build 方法中使用各種 “Hooks”，不再需要為了一些簡單的狀態或生命週期邏輯就創建一個 StatefulWidget 和與之關聯的 State 類別。
  • 邏輯複用
    • Hooks 本身就是可複用的邏輯單元。你可以創建自己的自訂 Hooks (custom Hooks) 來封裝常用的狀態邏輯或副作用（例如，處理動畫、訂閱數據流等），然後在不同的 HookWidget 中重複使用這些邏輯。
  • 更清晰的 Widget 結構
    • 相關的邏輯可以被組織在一起。例如，一個 AnimationController 的創建、監聽和銷毀邏輯可以放在同一個 useEffect Hook 中，而不是分散在 StatefulWidget 的 initState, didUpdateWidget, dispose 等不同方法裡。

常用的內建 Hooks

  • useState : 它讓你可以在函式型 Widget 中添加一個簡單的本地狀態。

  • useEffect : 用來處理副作用，例如數據獲取、手動操作 DOM (在 Flutter 中不直接操作 DOM，但類似概念如監聽器設定)、或設定計時器等。它類似於 StatefulWidget 中的 initState、didUpdateWidget 和 dispose 的組合。你可以指定依賴項，只有當依賴項改變時 useEffect 才會重新執行。它還可以返回一個清理函數，在 Hook 被銷毀時執行。

  • useMemoized: 用於緩存昂貴的計算結果。只有當它的依賴項改變時，才會重新計算。

  • 針對 Flutter 特性的 Hooks: flutter_hooks 還提供了很多方便的 Hooks 來管理 Flutter 特有的物件，例如：

    • useAnimationController(): 管理 AnimationController。
    • useTextEditingController(): 管理 TextEditingController。
    • useFocusNode(): 管理 FocusNode。
    • useStream() / useFuture(): 方便地訂閱 Stream 或 Future，並根據其狀態（載入中、有數據、有錯誤）自動重建 Widget。
