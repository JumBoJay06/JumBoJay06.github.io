基礎的基礎

Widgets (小部件)

  • 說明：在 Flutter 中，幾乎所有的東西都是 Widget。可以把 Widget 想像成應用程式介面 (UI) 的一個個積木。例如，一個按鈕是一個 Widget，一段文字是一個 Widget，甚至整個應用程式本身也是一個 Widget。
  • 重點：Widgets 描述了在給定其當前配置和狀態下，它們的視圖應該是什麼樣子。當 Widget 的狀態改變時，它會重新建構其描述，框架會將其與先前的描述進行比較，以確定對底層渲染樹進行最小更改，從而轉換到下一個狀態。

StatelessWidget (無狀態小部件)

  • 說明：這種類型的 Widget 在其生命週期內狀態不會改變。也就是說，一旦創建，它們的屬性就不能更改。

  • 用途：用於顯示靜態內容，例如標題、圖標或任何不需要內部狀態改變的 UI 元素。

StatefulWidget (有狀態小部件)

  • 說明：這種類型的 Widget 的狀態可以在其生命週期內改變。當狀態改變時，Widget 會被重建以反映新的狀態。

  • 用途：用於顯示動態內容，例如使用者輸入、動畫，或任何會因互動或時間而改變的 UI 元素。

  • 結構：StatefulWidget 由兩個類別組成：

    • 一個繼承自 StatefulWidget 的類別，它本身是不可變的，但它會創建一個 State 物件。
    • 一個繼承自 State 的類別，這個類別包含了 Widget 的可變狀態，以及 build() 方法。

佈局 Widgets (Layout Widgets)

  • 說明：Flutter 提供了多種 Widget 來幫助你排列和組織其他 Widgets。

  • 常見的佈局 Widgets：

    • Container: 一個方便的 Widget，可以組合繪製、定位和大小調整。你可以用它來設置背景顏色、邊距 (margin)、填充 (padding)、邊框等。
    • Row: 在水平方向上排列其子 Widgets。
    • Column: 在垂直方向上排列其子 Widgets。
    • Stack: 將其子 Widgets 堆疊在一起 (例如，一個 Widget 疊在另一個 Widget 之上)。
    • ListView: 一個可滾動的列表。
    • GridView: 一個可滾動的網格列表。
    • Expanded 和 Flexible: 用於在 Row 或 Column 中控制子 Widget 如何分配可用空間。
